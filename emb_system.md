| 模块 | 功能说明 | 项目对应/举例 | 关键接口/注意事项（选型 & 实现要点） |
|---|---|---|---|
| MCU / VCU（核心控制器） | 负责整车底层控制：驱动器组网管理、速度/扭矩指令下发、状态机（上电-预充-使能-运行-故障-安全停机）、故障处理与日志、与上位机/OBC通信 | 轮驱：8个轮驱驱动器控制；轮腿：抬腿伺服协调；急停/使能联锁 | 至少1路CAN（最好2路CAN）；若用RS485需485口；需要足够GPIO做使能/急停/故障输入；实时性（RTOS可选） |
| Flash / SRAM | Flash存固件、参数与标定值；SRAM运行时数据（协议栈缓冲、状态机、控制环数据） | 保存：轮子方向标定、驱动器ID映射、限流/加速度参数、CANopen节点配置等 | 需要参数掉电保存（内部Flash或外置EEPROM/FRAM）；支持固件升级（串口/USB/OTA） |
| 外设（GPIO/UART/SPI/I2C/ADC/PWM） | GPIO用于急停、驱动器使能、故障输入、限位；UART用于调试/上位机或部分设备；SPI/I2C接IMU等；ADC用于电压电流采样；PWM用于某些备用控制方式 | 轮驱：使能、故障灯/继电器；抬腿：限位开关；电源：电池电压采样 | 户外强干扰环境要注意：隔离、滤波、TVS、防静电；ADC采样要做分压与校准 |
| 轮驱电机（BLDC Traction Motor） | 将电能转换为轮端机械功；提供霍尔/编码器反馈以实现低速稳定与闭环控制 | 48V三相BLDC（你们已选型方向）；传感器：霍尔（后续可加编码器） | 要考虑：连续功率/连续电流、散热、IP防尘；霍尔低速抖动可用编码器改善 |
| 轮驱减速机构（Planetary Gearbox） | 降速增扭，匹配轮速与驱动力；与轮端机构集成以适应沙地重载与冲击 | 行星减速箱PLF/PZF系列（你们已在对比） | 不能让减速箱“硬吃”轮子径向/侧向冲击：建议轮端外置轴承支撑；关注额定输出扭矩与允许径向力 |
| 轮驱驱动器/伺服驱动（FOC Driver / Servo Drive） | 接收VCU指令（速度/扭矩/位置），输出三相UVW驱动电机；实现电流环/速度环（可选位置环）；提供过流/过压/欠压/过温保护与故障上报 | 你们对比过多款：含CAN/CANopen/RS485，多闭环等 | 必须有：输入电压范围覆盖48V电池满充；连续电流匹配电机；最好CAN/CANopen多节点；确认是否支持霍尔/ABZ编码器及协议文档 |
| 轮腿抬升执行器（Joint Servo Actuator） | 实现1DOF抬腿（主动悬挂/跨障）；与轮驱配合实现“抬起-落下-软着陆-限扭矩” | MF9025（你们已讨论）×8 | 常见为24V供电：需要48→24V DC-DC；最好支持位置控制+扭矩限制；关节结构需外置支撑/限位 |
| 传感器（状态+环境+运动） | 用于控制与安全：姿态IMU、轮速/里程计、温度、电流、电压；可选环境传感（温湿度/光照）用于作业评估 | 已提到IMU/里程计（可选）；驱动器常可回读电流/温度 | IMU常用SPI/I2C；轮速可来自驱动器反馈；关键是把“安全相关量”（电流/温度/电压）纳入故障策略 |
| 通信模块（车内总线 + 远程链路） | 车内：VCU与8个驱动器/伺服的实时通信；车外：与上位机/遥控/上层计算单元通信 | 车内建议CAN/CANopen（已多次讨论）；也可RS485(Modbus) | 总线必须有终端电阻、线缆屏蔽与接地；确认CANopen是否提供EDS/对象字典或Modbus寄存器表 |
| 电源管理（低压电源树） | 将48V主电转换为24V/12V/5V，供给控制器、伺服、传感器；监控电压并在异常时安全降级/停机 | 48V→24V给MF9025（若需要）；48V→12/5V给VCU/传感器 | DC-DC要留足裕量；做输入浪涌/反接保护；低压电源建议分区保险，避免单点短路拖死全车 |
| 高压/大电流配电与安全（新增） | 48V主回路的安全与上电流程：主保险、接触器、预充、支路保险、急停硬切断，保证故障时可控停机 | 前面提到急停、配电、主电源等（需要在系统图补齐） | 8WD电流很大：必须做PDU/母排/支路保险；急停建议硬件级切断接触器或驱动使能 |
| 线束/连接器/防护（新增） | 将电源、UVW、反馈、总线可靠地连接到每个轮模块；提供防尘、防水、抗振与应力释放 | 你们担心“8套很麻烦”，线束设计是关键工作量 | 选IP等级连接器；动力线与信号线分开走；屏蔽与接地；腿部运动建议拖链/护套；做好线束编号与一致化 |
| 软件架构（新增） | 通信协议栈、状态机、速度控制、限流/限扭矩、轮间同步、故障处理、日志与参数管理 | 之前讨论了“bring-up台架”“8轮一致性”“限扭矩/软启动” | 建议按模块分层：Drivers(BSP)→Comms→MotorCtrl→VehicleCtrl→Safety；提供可复现的调参流程与记录 |
| 调试与测试（新增） | 单轮台架验证、8轮联调、故障注入测试（欠压/过流/断线）、温升测试、沙尘环境可靠性检查 | 之前建议先做单轮台架再复制到8轮 | 必备工具：USB-CAN/USB-485、示波器、电流钳、热像仪（可选）；建立“测试用例清单” |
